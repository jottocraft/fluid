settingsHandling User Preferences & Settings

Creating additional settings in your website can be painful. You have to find space for the switch, and when you do, you have to ensure the switch always represents the setting, even if it is changed elsewhere. This can add up to a lot of unnessasary code. With Fluid UI v4, there are 3 new functions to make settings and preferences easier to manage.

The name of the setting can be any string starting with "pref-". Your setting's name is used as a CSS class for elements on your page that represent the setting (like a switch or button for a boolean, or a button row or list for multiple choice)

The button row demos above also act as preferences demos. Prefrence changes sync across tabs, so if a user has multiple tabs of your site open and changes a prefrence on one of them, it will apply to all tabs.


Setting a preference (If the prefrence is a boolean, call fluid.set without a value to toggle it)

fluid.set([setting string], [value string])

Getting a preference

fluid.get([setting string])

Listening to a preference change (use event.detail for the new value)

document.addEventListener(pref-[setting string], [callback function])

If your value is a boolean, you can use a switch to represent the setting. When using an element to represent a setting, add the setting string to the class list. The default value should be represented with the active class for true and no active class for false.

<div class="switch [setting]"><span class="head"></span></div><div class="label">My new setting</div>

If your value is a string, you can use a button row to represent the setting. When using an element to represent a setting, add the setting string to the button row's class list, and the value string to each of the button's class lists. The default value of the setting should be represented by adding the active class to the respective child button.

            <div class="btns row [setting string]">
  <button onclick="fluid.set(...)" class="btn [value string]">Option A</button>
  <button onclick="fluid.set(...)" class="btn [value string]">Option B</button>
  <button onclick="fluid.set(...)" class="btn [value string]">Option C</button>
</div>

Elements that are properly configured to toggle or change a setting will automatically do so without needing any extra code. You do not need to call fluid.set every time a switch is toggled if it is configured to toggle the setting. Remember to call fluid.init to add the event listeners if you add an element representing a setting after the page loads.

Themes

With the introduction of Fluid Flex in Fluid UI v3, Fluid UI ships with several standard and colorful themes. Standard themes, or Fluid Themes, have full control over the UI, with support for complex color palettes and additional control over the UI, including font family, border radius, shadows, and more. Fluid Themes should be used when creating a new theme on your site that you want to be fully supported and accessible.

Fluid Flex themes are lighter, simpiler themes, designed for special events, easter eggs, or just for fun. To change themes in Fluid UI, you need to use fluid.theme (Changing the theme manually can result in issues with your site). To change the default theme on your site, you can change the body element's class. To allow a user to change their theme, you can use a Fluid UI theme selector, or create your own UI that calls fluid.theme. All sites using Fluid UI cannot disable theming completely. If you do not want users changing their theme, do not provide a theme changing UI. However, users can still manually call fluid.theme or override the theme variables on your site.

Fluid UI includes the themes listed below:

  • Special Theme (time-based): Automatic (ID: "auto")
  • Fluid Theme (white): Default Light (ID: "light")
  • Fluid Theme (dark gray): Standard Dark (ID: "dark")
  • Fluid Theme (black): Midnight Black (ID: "midnight")
  • Fluid Theme (dark purple): Tome (ID: "tome")
  • Fluid Theme (dark red): Nitro (ID: "nitro")

Get the current theme (When using the auto theme, this will return the theme that was automatically chosen)

fluid.theme()

brightness_autoAutomatically set theme (default)

During Standard Time:                    Light theme: 6AM - 5PM    Dark theme: 6PM - 5AM
During Daylight Savings Time:     Light theme: 5AM - 6PM    Dark theme: 7PM - 4AM

If system-wide dark mode is enabled and the user is using a supported browser, dark theme applies by default

JavaScript function

fluid.theme("auto");
Auto theme is defaultly enabled on this site, so changes may not appear when running the demo. Try the theme selector demo below if nothing happens.



buildSet the default theme

You can set the default theme on your site if you want to disable dark mode, always use dark mode, or use any theme you want by default. To do this, you need to change the classes added to the body element. If you do not want users changing the theme on your site, do not provide a theme selection UI. You should never change the theme classes on the body element after the page loads as that might cause inconsistencies between the Fluid UI theme selector, theme prefrence in localStorage, and the theme displayed on the site. Always use fluid.theme after page load to set the theme.

You can set the default theme by adding the theme ID to the body's class. If setting a custom theme that starts with "dark" (white text on a dark background), make sure to also add the dark class alongside the custom theme's ID. If the theme is a Fluid Flex theme, you also need to add the flex class. If you experience issues setting the default theme, remove all classes from the body element, call fluid.theme, and check the classes added to the body element to see what you need to add.

<body class="themeID">

Set theme with JavaScript. The theme variable can either be "toggle" for toggling between light and dark or a theme ID. Set the second parameter to true to prevent Fluid from saving the theme as the user preference and applying it on reload (for demos or if you have your own way of storing users' theme preferences)

fluid.theme(theme, [override theme saving]);

arrow_drop_down_circleTheme Selector

The theme selector shows themes that you configure using the fluidThemes variable. The theme selector can show as many pages as you want, but it is reccommended to keep the length under 4 pages. The first page always shows Light/Dark/Auto themes. The other pages can be configured to show Fluid Flex themes. fluidThemes should be an array of arrays. Each array in fluidThemes represents a page. Each page is an array of the themeIDs you want to enable. Custom themes should be objects with the name and id properties, with the theme's display name and themeID, and optionally a material icon string in the icon property. There is no way to disable the default light/dark/auto themes page. All theme selectors must include at least light, dark, and auto.

The rainbow theme selector can be enabled by adding "rainbow" the array of a page (adding "rainbow" to a page causes that page to show rainbow themes only. You cannot mix the rainbow themes page with other custom themes). Note that the rainbow themes are intended to be used for fun and are not officially supported.

Add this code before loading fluid.js to specify what themes you want to be shown in the theme selector

fluidThemes = [ [ "midnight", "nitro", "tome" ], [ "rainbow" ] ]

textureFluid UI
          Acrylic
        

        
Fluid UI now has a transparency effects with Fluid UI Acrylic. Acrylic is enabled by default on cards with
          sidenavs, sidebars, toolbars, and navbars. You can also use Acrylic with any element by adding the
          "acrylicMaterial" class. Make sure you test custom Acrylic implementations, and make sure you don't overuse
          Acrylic on your site.

        
Acrylic is designed to work on any background (images, solid colors, and even if there is no background) and
          to provide enough contrast to allow most colors to be displayed and legible. Custom Fluid themes can set the
          acrylic tint color by setting the --acrylicBase CSS variable. Fluid Flex themes do this for you automatically.
          You can provide a solid color and have Fluid UI calculate the opacity for you or provide a color with an
          opacity of at least 70%. When setting the Acrylic tint color, make sure that it also works over your page's
          background since Acrylic should be able to be used without a custom background.

        
Acrylic can be completely disabled on your site by adding the "disableAcrylic" class to the body element. If
          Fluid UI couldn't calculate the acrylic variables and during page load before the variables are calculated,
          acrylic will fallback to --acrylicBase or --elements.

webScreens

          
Fluid UI Screens is a flexible way to manage multiple different screens on the same page. You can define a
            function that will load each screen, then use Fluid UI to change screens. Fluid UI will save the screen in
            the URL so that it will show the screen on reload and so the user can bookmark the screen. Fluid UI screens
            also supports the browser back button.

          
Screen names and the screen ID parameter cannot include slashes (/) or periods (.)

          

          
Define a screen:

          
fluid.screens.folderScreen = function(id, entry) {
  //screens have an optional ID for whatever the screen is showing (in this example, the ID would be the ID of the folder the user is navigated to)
  //the screen ID must be a string and able to fit into the URL
  //the entry parameter is a boolean specifying whether this screen is the first to load on the page

  //code that will show this screen
};

          

          
Define the default screen:

          
fluid.defaultScreen = "homeScreen";

          

          
Open a screen:

          
fluid.screen("folderScreen", "folder18232372");

          

          
When your site is done loading and you want to show the user the screen from the URL or the default screen,
            run fluid.screen with no parameters:

          
fluid.screen();
