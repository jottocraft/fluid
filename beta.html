Twemoji

Fluid UI automatically parses emoji on your site on load into Twitter emoji 🙂. Add the "notwemoji" class to the body if you want to use native emoji instead.

<body class="notwemoji">

If your site renders emoji after load, tell Twemoji to re-parse

twemoji.parse(document.body);

Button row

Buttons can be placed on rows to create a selector. Plus, they handle the active class with no additional code required! You can add the active class to the default button, and handle the clicks yourself to get the data.

Because buttons are inline, if you want to use buttons in a row normally only the row class 

<div class="btns row">
<button class="btn">Option A</button>
<button class="btn">Option B</button>
<button class="btn">Option C</button>
</div>

Dark mode

Fluid UI ships with a dark mode, which you can toggle easily with only one line of code, or use a CSS class. Dark mode currently only supports components from Materialize. To use your own elements with dark mode, read the instructions in the fluid.css file.

Check if dark theme is enabled

fluid.isDark()

Automatically set theme based on time (Reccommended)
Light mode: 6AM - 6PM Dark mode: 7PM - 5AM

Enable with CSS

<body class="auto">

Enable with JavaScript

fluid.auto();
Auto theme is defaultly enabled on this site, so changes may not appear when running the demo



Set Fluid's theme manually

Set with CSS (Dark class sets dark mode, no dark class sets light mode)

<body class="dark">

Set with JavaScript (Toggle by running without parameter)

fluid.dark(true);

Fluid Cloud

Fluid Cloud is a Fluid UI module that supports Fluid UI 3 or later versions. Fluid Cloud lets you configure alerts, which could be used for events or updates. Fluid Cloud can also be used to set a splash screen in case of an unexpected outage on your site.

All of the site configuration (active splash screen and alert text) is public

You can go to the Fluid Cloud console by clicking on the cloud icon at the top of the page, or by going to fluid.js.org/cloud


Initilization code

Give your site a unique (for your account, not global) Fluid Cloud project ID. You will be given your Fluid Cloud user ID when you login to the cloud console.


fluid.cloudConfig variable (Make sure it is defined before loading cloud.js):

<script src="https://www.gstatic.com/firebasejs/5.1.0/firebase.js"></script>
<script>
fluid.cloudConfig = {
  cloudUser: "myuserid",
  cloudProject: "site"
}
</script>
<script src="fluid/cloud.js"></script>

If your site has a web app manifest, you can define the cloudProject variable there. You still need to make sure to include both JavaScript files, however.

Web app manifest:

{
"name": "My site",
"description": "Cool site",
...
"cloudUser": "myuserid",
"cloudProject": "site",
...
}

Navigation

Fluid UI also offers simple navigation bars. You can add the active class to any one of these to show that it is active. Add the active class to the nav div itself to automatically give buttons the active class.

You can add text in either navigation bar style for clarity by putting a span next to the icon. See the navbar style for an example.

Toolbar

Toolbars are small and simple (The default navigation on this site is a toolbar)

          <div class="nav right">
        <ul>
            <li class="active">
            <a href="#"><i class="material-icons left">info</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">help</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">build</i></a>
            </li>
        </ul>
</div>

You can also move it to the left by changing its class: 

<!-- Left alignment -->
<div class="nav left">

You can make the navigation sticky (always stays at the top no matter where the user is scrolled) by adding the sticky class: 

<!-- Sticky nav -->
<div class="nav right sticky">

Navbar

Navbars take up the entire top of the page and let you add your own title and logo

    <div class="nav navbar">
   <img src="https://fluid.js.org/icon.png" />
   <h4>Fluid UI</h4>
    <ul class="right">
        <li class="active">

Sections and Cards
Sections

Using a section to seperate different components on a page in your site is reccommended. Sections are easy to setup, and provide contrast against the background.

<div class="section">
    <div class="header">
      <i class="material-icons">note</i>
      <h4>Demo</h4>
    </div>
    <div class="body">
    Hello!
    </div>
</div>

Add the "collapse" class to the section to indicate that it can be collapsed

<div class="section collapse">

Add the "collapsed" class to the body of the section to have it defaultly collapsed

<div class="body collapsed">

Click the try button below to give all sections on this page the collapse class. If your page dynamically changes the collapse class, make sure to run fluid.init() to initialize all of them.




Cards

If you want a section with a 3D effect, use a card. However, cards do not work on mobile, and can cause lag if you use too many of them.

<div id="democard" class="card">
  <!-- Card Content -->
  </div>

Add the "close" class to make the card hidden by default

<div id="democard" class="card close">

Add the "focus" class to make the card act as an alert and blur the background. Clicking on the background will also close the card!

<div id="democard" class="card focus close">

Show a card:

fluid.cards(card-id)

To close cards (Use .card.focus for focused cards):

fluid.cards.close(elementSelector);

Splash Screens and Alerts

Show your div as a splash screen with fluid.splash(). Use the title attribute to specify the title of the splash screen, and add the splash class.

<div class="Splash Demo" title="Splash Demo">
fluid.splash(element)


You can also use a focus card as a modal in Fluid UI. Just use fluid.modal() to open the card and it will disable scrolling and act as an alert.

fluid.modal(element)
