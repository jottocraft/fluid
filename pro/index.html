Fluid Pro



Fluid Pro was made as a more advanced version of Fluid UI. It is way more complex than Fluid UI. Before using Fluid Pro, make sure you already know CSS well and are fully familiar with the basics of Material Web Componenets and Fluid UI.

Fluid Pro is seperate from Fluid UI because it is not reccommended for most users, is probably more buggy, and will be updated less often.

Integrated dark mode


Fluid supports a dark mode, which you can enable with only one line of code. Dark mode only supports elements from Material Web Componenets. To integrate other elements, read the notes section.


1

Toggle dark mode with JavaScript

fluid.dark();



2

Enable dark mode with CSS by adding the dark class to the body

<body class="dark">

Card UI


If you don't nessisarily need a full page for content, but you don't want it being part of the page itself, make it a card


  <div id="democard" class="card">
  <!-- Card Content -->
  </div>

Add the "close" class to make the card hidden by default

<div id="democard" class="card close">

Add the "focus" class to make the card float at the top of the page, scroll the user to the top, and blur the background. Clicking on the background will also close the card!

<div id="democard" class="card focus close">

Because focus cards blur the background and close the card on click, you will need to create a content div as the parent for every part of your page, excluding the focus card itself (It doesn't matter where the card is)

<div class="content">

To show a card:

fluid.cards(card-id)

To close a card (where card is #id, .class, etc.):

fluid.card.close("card");

To close all cards:

fluid.cards.close();

To close only focus cards:

fluid.cards.close.focus();

Other Elements

Everything else that isn't specified here can be found at Material's documentation. Fluid Pro is built off of Material Web Componenets 0.30.0.


Colors used in Fluid can be found here.


Everything listed in Material's docs should work as expected in the light theme, however some componenets may need to be manually optimized for the dark theme 



/* Syntax

  Light mode
  body:not(.dark) *element(s)* {
    /* Your element's light theme style */
  }

  Dark mode
  body.dark *element(s)* {
    /* Your element's dark theme style */
  }

*/
