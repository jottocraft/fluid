Button row

Buttons can be placed on rows to create a selector. Plus, they handle the active class with no additional code required! You can add the active class to the default button, and handle the clicks yourself to get the data. You can also add the "multiple" class to allow multiple to be selected at once, making it act as checkboxes.



<div class="btns row">
  <button class="btn">Option A</button>
  <button class="btn">Option B</button>
  <button class="btn">Option C</button>
</div>

Emoji

Fluid UI automatically parses emoji on your site on load into Twitter emoji 🙂. Add the "notwemoji" class to the body if you want to use native emoji instead.

<body class="notwemoji">

If your site renders emoji after load, tell Twemoji to re-parse

twemoji.parse(document.body);

Lists

Themes

With the introduction of Fluid Flex in Fluid UI 3.0.0 beta 5, Fluid UI ships with several standard and colorful themes. You can use any of the included themes (list below), or make your own theme. The Fluid Flex update completly reworks the Fluid UI theme engine, and changes the default theme from light to auto. You can change the theme by adding the theme ID as a class to the body element, or using the fluid.theme JavaScript function. You can also use an included theme selector or make your own theme selection UI.

  • Default Light (ID: "light")
  • Standard Dark (ID: "dark")
  • Midnight Black (ID: "midnight")
  • Nitro (ID: "nitro")
  • Aqua (ID: "aquatic")

Get the current theme (When using the auto theme, this will return the theme that was automatically chosen)

fluid.theme()

Automatically set theme (Default)
Light mode: 6AM - 5PM Dark mode: 6PM - 5AM

JavaScript function

fluid.theme("auto");
Auto theme is defaultly enabled on this site, so changes may not appear when running the demo. Manually change the theme and come back here if it doesn't do anything.



Manually set theme

You can set the theme to whatever you want. You can choose to do this to change the default theme and still allow users to change their own theme, or you can make your site fixed with a single theme. The user's prefrences (from cookies) will always override the default theme unless the force class is added to the body (this tells Fluid UI to ignore user prefrence completly). If you don't have a theme selection UI, it is reccommended to not add the force class to allow advanced users to manually enable a theme you don't support.

<body class="themeID">

Set theme with JavaScript. The theme variable can either be "toggle" for toggling between light and dark or a theme ID. Set the second parameter to true to prevent Fluid from saving the theme as the user prefrence and applying it on reload (for demos or if you have your own way of storing users' theme prefrences)

fluid.theme(theme, [override theme saving]);

Theme Selector

The theme selector shows themes that you configure using the fluidThemes variable. The theme selector can show up to 4 pages. 1: Light/Dark/Auto 2: Midnight Black/Nitro/Aqua 3: Any custom themes 4: All rainbow colored themes (not reccommended). fluidThemes should be an array with each of the themeIDs you want to enable. Custom themes should be objects with the name and themeID properties, and optionally a material icon in the icon property. The rainbow theme selector can be enabled by adding "rainbow" to the array. There is no way to disable the light/dark/auto themes. Pages 1, 2 and 4 are demoed below.


<div class="btns row themeSelector"></div>

Custom themes should be prefixed with "light" or "dark" depending on what theme you want to use as the base. This also allows Fluid UI to recognize the theme. Make sure to include the flex prefix when using fluid.theme to apply your custom theme.

          
body.darkMyTheme {
      /* Fluid Flex (UI colors) */
      --flex-light: rgb(149, 14, 14);
      --flex-bg: rgb(127, 1, 1);
      --flex-text: white;
      --flex-layer1: rgb(129, 7, 7);
      --flex-layer2: rgb(104, 0, 0);
      --flex-layer3: #5c0101;

      /* Brand Colors (Active elements and other colored accents. Flex themes do not need to include this but it is reccommended.
      You can also use these by themselves without a flex theme to apply to the default light/dark themes.) */
      --theme-color: #0bdd77;
      --theme-color-outline: #3396e6C0;
      --theme-text-color: white;
}

Navigation

              

              

      

        Fluid UI also offers simple navigation bars. You can add the active class to any one of these to show that it is active. Add the active class to the nav div itself to automatically give buttons the active class.
      

      
You can add text in either navigation bar style for clarity by putting a span next to the icon. See the navbar style for an example.

Sections and Cards





  
  
Sections

  
Using a section to seperate different components on a page in your site is reccommended. Sections are easy to setup, and provide contrast against the background.


<div class="section">
    <div class="header">
      <i class="material-icons">note</i>
      <h3>Demo</h3>
    </div>
    <div class="body">
    Hello!
    </div>
</div>



Add the "collapse" class to the section to indicate that it can be collapsed.

<div class="section collapse">




Add the "collapsed" class to the body of the section to have it defaultly collapsed

<div class="body collapsed">




Click the try button below to give all sections on this page the collapse class. If your page dynamically changes the collapse class, make sure to run fluid.init() to initialize all of them.








Cards


If you want a section with a 3D effect, use a card. However, cards do not work on mobile, and can cause lag if you use too many of them.


<div id="democard" class="card">
  <!-- Card Content -->
  </div>





  Add the "close" class to make the card hidden by default


  
    <div id="democard" class="card close">
  
  

    Add the "focus" class to make the card act as an alert and blur the background. Clicking on the background will also close the card!
  

    
      <div id="democard" class="card focus close">
    
  

    Show a card:
  

    
      fluid.cards(card-id)
    
    

      To close cards (Use .card.focus for focused cards):
    

      
        fluid.cards.close(elementSelector);

Splash Screens and Alerts





  
  
Splash Screens


Show your div as a splash screen with fluid.splash(). Use the title attribute to specify the title of the splash screen, and add the splash class.

  <div class="Splash Demo" title="Splash Demo">
  

  fluid.splash(element)
  

  
  


  
  
Toasts

  
Use fluid.toast, and include a material icon string and the toast's message

  fluid.toast(materialIcon, message)
  

  
  


  
  
Modals

  
You can also use a focus card as a modal in Fluid UI. Just use fluid.modal() to open the card and it will disable scrolling and act as an alert.

  fluid.modal(element)
