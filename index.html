Button row

Buttons can be placed on rows to create a selector. Plus, they handle the active class with no additional code required! You can add the active class to the default button, and handle the clicks yourself to get the data.

Because buttons are inline, if you want to use buttons in a row normally only the row class 

<div class="btns row">
<button class="btn">Option A</button>
<button class="btn">Option B</button>
<button class="btn">Option C</button>
</div>

Dark mode

Fluid UI ships with a dark mode, which you can toggle easily with only one line of code, or use a CSS class. Dark mode currently only supports components from Materialize. For additional compatibility notes, read the bottom of the page.


1

Toggle dark mode with JavaScript

fluid.dark();



2

Enable dark mode with CSS by adding the dark class to the body

<body class="dark">

Navbar

Fluid UI also offers a simple and modern style navigation bar. You can add the active class to any one of these to show that it is active. Add the active class to the nav div itself to automatically give buttons the active class. 

          <div class="nav right">
        <ul>
            <li class="active">
            <a href="#"><i class="material-icons left">info</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">help</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">build</i></a>
            </li>
        </ul>
</div>

You can also move it to the left by changing its class: 

<!-- Left alignment -->
<nav class="left">

layers

Section and Card UI

Section

Using a section to seperate different components on a page in your site is reccommended. Sections are easy to setup, and provide contrast against the background.

<div class="section">
    <div class="header">
      <i class="material-icons">note</i>
      <h4>Demo</h4>
    </div>
    <div class="body">
    Hello!
    </div>
</div>

Cards

If you want a section with a 3D effect, use a card. However, cards do not work on mobile, and can cause lag if you use too many of them.

<div id="democard" class="card">
  <!-- Card Content -->
  </div>

Add the "close" class to make the card hidden by default

<div id="democard" class="card close">

Add the "focus" class to make the card act as an alert and blur the background. Clicking on the background will also close the card!

<div id="democard" class="card focus close">

Because focus cards blur the background and close the card on click, you will need to create a content div as the parent for every part of your page, excluding the focus card itself (It doesn't matter where the card is)

<div class="content">

Show a card:

fluid.cards(card-id)

Close a card (where card is #id, .class, etc.):

fluid.card.close("card");

Close all cards:

fluid.cards.close();

Close only focus cards:

fluid.cards.close.focus();

note

Notes

All the files in Fluid UI are made to be easily modifiable. If you made a modification you like, feel free to make a PR on GitHub.

Because Fluid UI is modded Materialize, you can find use most Materialize elements (and they all support dark mode!). Materialize's documentation. Fluid UI is built off of Materialize 0.100.2.

Colors used in Fluid UI can be found here. (I use this myself for development so it's always up to date)

Most things should work as expected from Materialize's code. However, some things may be unstable, or not work altogether. Contact me with any issues you think should be added to this list.

Materialize components Copyright (c) 2014-2018 Materialize


Contact

Email jotto3053@gmail.com to contact me


Unsupported

Because some Fluid UI components override / overlap with Materialize components, some Materialize components aren't functional
  • Materialize input labels, character counters, and error / success messages
  • Materialize cards
  • Materialize sections
  • Anything not included in Materialize v0.100.2

Install Fluid UI

Make sure Fluid UI is in the /fluid directory

          
<!-- Fluid UI -->
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
<link type="text/css" rel="stylesheet" href="/fluid/fluid.css" media="screen,projection"/>
<meta name="viewport" content="width=device-width, initial-scale=1.0"/>
<script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/3.3.1/jquery.min.js"></script>
<script type="text/javascript" src="/fluid/fluid.js"></script>
