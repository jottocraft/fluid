Buttons with the btn class will also get the waves-effect class. To override this, just add the override class. It is still reccommended to manually add the class, in case of a bug or error.




<button class="btn">Hello!</button>


Integrated dark mode


Fluid supports a dark mode, which you can enable with only one line of code. Dark mode currently only supports components from Materialize in the CSS and Components tab. The final release will allow all components from Materialize. For additional compatibility notes, read the bottom of the page.


1

Toggle dark mode with JavaScript

fluid.dark();



2

Enable dark mode with CSS by adding the dark class to the body

<body class="dark">


Navigation bar


Setup a navigation bar easily (Other Materialize navbar styles may not work as expected): 

          
<nav class="right">
    <div class="nav-wrapper">
        <ul>
            <li>
            <a href="#"><i class="material-icons left">info</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">help</i></a>
            </li>

            <li>
            <a href="#"><i class="material-icons left">build</i></a>
            </li>
        </ul>
    </div>
</nav>

You can also move it to the left by changing its class: 


<!-- Left alignment -->
<nav class="left">


Easy to use loader


Setup a loader without having to run initialization code:

<loader></loader>

Show the loader (The loader cannot have any display style overrides. Use the hidden class instead to hide by default)

fluid.load(true);

Hide the loader

fluid.load(flase);

Card UI


If you don't nessisarily need a full page for content, but you don't want it being part of the page itself, make it a card


  <div id="democard" class="card">
  <!-- Card Content -->
  </div>

Add the "close" class to make the card hidden by default

<div id="democard" class="card close">

Add the "focus" class to make the card float at the top of the page, scroll the user to the top, and blur the background. Clicking on the background will also close the card!

<div id="democard" class="card focus close">

Because focus cards blur the background and close the card on click, you will need to create a content div as the parent for every part of your page, excluding the focus card itself (It doesn't matter where the card is)

<div class="content">

To show a card:

fluid.cards(card-id)

To close a card (where card is #id, .class, etc.):

fluid.card.close("card");

To close all cards:

fluid.cards.close();

To close only focus cards:

fluid.cards.close.focus();

Notes


All the files in Fluid are made to be easily modifiable. If you made some cool modification, fork the project on GitHub and send it over to me.


Other Elements

Everything else that isn't specified here can be found at Materialize's documentation. Fluid CSS is built off of Materialize 0.100.2.


Colors used in Fluid can be found here. (I use this myself for development so it's always up to date)


Most things should work as expected from Materialize's code. However, some things may be unstable, or not work altogether. Note that Fluid is still in beta, so this may be inaccurate. Contact me with any issues you think should be added to this list.



About / Contact

Email jotto3053@gmail.com for anything you want to ask. Fluid cost nothing to make (excluding my time), and will always be ad-free and open to everyone. I made Fluid after I made Teleport and wanted to make the design and dark mode easy to use.


Unsupported

  • Materialize input labels
  • Materialize's cards
  • Materialize input character counter (only works in textareas)
  • Materialize input Error or Sucess messages



Install Fluid

download   |   v1.1


Install using Fluid's files

Download Fluid's files to guarentee availability, and make sure your site is optimized before updates.

          
<!-- Fluid UI -->
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
<link type="text/css" rel="stylesheet" href="fluid/fluid.css" media="screen,projection"/>
<meta name="viewport" content="width=device-width, initial-scale=1.0"/>
<script type="text/javascript" src="https://code.jquery.com/jquery-3.2.1.min.js"></script>
<script type="text/javascript" src="fluid/fluid.js"></script>


Install using the CDN

Install using Fluid's CDN if you do not want to store Fluid's files on your website. This method will update automatically, and availability is not guarenteed. This method is not recommended.

          
<!-- Fluid UI -->
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
<link type="text/css" rel="stylesheet" href="https://cdn.rawgit.com/jottocraft/fluid/master/fluid.css" media="screen,projection"/>
<meta name="viewport" content="width=device-width, initial-scale=1.0"/>
<script type="text/javascript" src="https://code.jquery.com/jquery-3.2.1.min.js"></script>
<script type="text/javascript" src="https://cdn.rawgit.com/jottocraft/fluid/master/fluid.js"></script>
